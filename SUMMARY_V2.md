### 新数据格式示例

```json
{
  "id": 1,
  "text": "此次培训取的了良好效果",
  "original_text": "此次培训取得了良好效果",

  "has_error": true,
  "error_types": ["typos"]

```

### 数据格式优势

✅ **JSONL 格式**
- 流式读取，内存友好
- 易于追加新数据
- 并行处理支持
- 行级别的错误容错

✅ **完整的错误标注**
- 记录原始文本和错误文本
- 详细的错误类型分类
- 精确的位置信息
- 便于错误分析和改进

## 🎯 错误类型覆盖

训练数据涵盖 **6 大类** 公文常见错误：

### 1. 的地得混用 ⭐
```
错误示例：走地很快 → 走得很快
          缓慢的前进 → 缓慢地前进
```

### 2. 常见错别字 ⭐⭐
```
错误示例：取的 → 取得
          近一步 → 进一步
          任真 → 认真
          经神 → 精神
          祖织 → 组织
```

### 3. 非正式用语 ⭐⭐
```
错误示例：关与 → 关于
          贯切 → 贯彻
          监于 → 鉴于
```

### 4. 同音字混淆 ⭐
```
错误示例：制订 ↔ 制定
          截至 ↔ 截止
          反应 ↔ 反映
```

### 5. 标点符号 ⭐
```
错误示例：通知如下。 → 通知如下：
          研究决定。 → 研究决定，
```

### 6. 量词搭配 ⭐
```
错误示例：一个文件 → 一份文件
          三个措施 → 三项措施
```

## 💻 使用方法

### 1. 快速演示

```bash
python inference_v2.py
```

会自动运行 4 个演示案例，然后进入交互模式。

### 2. 测试集评估

```bash
python inference_v2.py --test
```

在 200 个测试样本上评估模型性能。

### 3. 在代码中使用

```python
from inference_v2 import DocReviewer

# 初始化
reviewer = DocReviewer("models/best_model_v2")

# 检测文本
text = "此次培训取的了良好效果，达到了预期目标。"
errors = reviewer.review(text)

if errors:
    for err in errors:
        print(f"错误位置: {err['start']}-{err['end']}")
        print(f"错误文本: {err['text']}")
```

### 4. 重新训练

```bash
# 生成新数据
python generate_data_v2.py

# 训练模型
python train_v2.py
```

## 🎓 最佳实践参考

本项目采用的行业最佳实践：

### 1. 数据格式
✅ **JSONL** - NLP 任务标准格式
- 每行一个 JSON 对象
- 便于流式处理大规模数据
- 易于版本控制和 diff

### 2. 标注方案
✅ **BIO 标注** - 序列标注标准
- B (Begin) - 实体开始
- I (Inside) - 实体内部
- O (Outside) - 非实体

### 3. 评估指标
✅ **多维度评估**
- Precision (精确率) - 检测的准确性
- Recall (召回率) - 检测的覆盖率
- F1 Score - 平衡指标
- Accuracy - 总体准确率

### 4. 训练技巧
✅ **标准优化方法**
- AdamW 优化器
- 学习率预热
- 梯度裁剪
- 早停机制（实际保存最佳模型）
